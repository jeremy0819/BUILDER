# 判斷力外化（Judgment Rubrics）

> 讀者：弱模型 session。每條 rubric ＝ 判準＋一正例＋一反例。
> 用法：遇到「該不該／算不算／要不要」的猶豫時，先查本檔再行動。查不到的情況 → 按 R3 停下問使用者。

---

## R1 何時升級模型

**判準**：滿足任一 → 升級（路徑見 MODEL_DISPATCH §4）：
- 同一驗收條件下已失敗達門檻次數（haiku 1 次／sonnet 連 2 次）。
- 任務涉及「多份文件互相矛盾，需要裁決」而非「照文件執行」。
- 任務涉及公式推導、法規解讀、金額計算的**對錯判斷**（不是照抄既有公式）。

**正例**：sonnet 兩次都把「陽台超出」用 FA 總量法算（高估 10 倍），第三次不再重試，
帶著兩次錯誤輸出＋REVIEW.md 的教訓段升級 opus。
**反例**：改 README 錯字失敗一次（其實是檔案路徑打錯），就升級 opus——這是路徑問題，
不是能力問題，修正路徑重試即可。

## R2 何時算真的完成（Definition of Done）

**判準**：以下全過才能說「完成」，缺一項就如實說「做完但未驗 X」：
1. 驗收指令實際跑過且輸出符合（不是「應該會過」）。
2. RE-DCF 動到公式 → `pytest` 全綠；動到 L6 → 重跑 `python make_template.py` 同步 Excel 範本。
3. 動到 UI → 真的啟動（`streamlit run app.py` 或開 HTML）看到畫面正常，不能只看測試。
4. 動到檔案 → fresh-context agent read-back 通過。
5. CHANGELOG／版本號按該 repo 規則更新。

**正例**：改了 `core/finance.py` → pytest 綠 → 重跑 make_template.py → 開 Streamlit 載入案例C
看 Tab⑤ 數字正常 → 更新 CHANGELOG → 回報時附各步輸出。
**反例**：改了 app.py，pytest 綠就回報完成——pytest 根本不測 UI，v4.8 就是這樣整頁崩掉的。

## R3 何時停下來問使用者

**判準**：碰到以下任一，先停，附上你的建議選項再問：
- 變更 `schemas/project_schema.json`（凍結中）或黃金測試**期望值**。
- 變更費率常數（設計監造 5%、管理費 5% 等——這些是真實案校準成果，不是普通參數）。
- 刪除檔案、改 `.gitignore`、任何 LICENSE／授權決定。
- 涉及真實案件資料的處理（哪怕只是「順手」把一個地名寫進註解）。
- 部署設定（Streamlit Cloud 進入點、GitHub Pages）。
- 兩份權威文件對同一事實說法不同，且影響當前動作。

**正例**：測試期望值 7616.20 與新算出的 7616.00 差 0.2——不是改期望值讓測試過，
而是停下來問：這是公式錯了還是期望值本來就含邊界設計？
**反例**：為了讓匯入功能動起來，「順手」在 schema 加一個 optional 欄位——凍結就是凍結，
optional 也不行（P1-4 驗收是位元組級 hash 不變）。

## R4 方向錯了的訊號（該換路，不是再重試）

**判準**：出現任一訊號 → 停止當前路線，回頭重新讀需求或問使用者：
- 為了讓方案成立，需要違反紅線（把公式搬進前端、解凍 schema、引入 bundler）。
- diff 越修越大：第二輪修正的改動量超過第一輪。
- 需要新增依賴套件才能解一個小問題。
- 修 A 壞 B、修 B 壞 A 循環出現（同一組測試紅綠交替兩輪）。
- 發現自己在改「不在任務範圍內的檔案」以繞過障礙。

**正例**：evaluator 匯入功能要顯示新欄位，發現 schema 沒有 → 正解是記 backlog 等 schema 2.0，
而不是讓前端自己算（違反 SSOT）。
**反例**：測試紅了兩次，第三次把測試 skip 掉——這不是換路，是把訊號滅音。

## R5 品質底線怎麼驗（每次 push 前的最小檢查）

依 repo 執行，任何一條紅 → 不 push：

**RE-DCF-Tool（合併後 core/redcf/）**
```bash
pytest                                                    # 全綠
grep -rn "竹蓮\|安和\|安民\|中正\|龜山" --exclude-dir=.git .   # 零命中
sha256sum schemas/project_schema.json                     # 凍結期須等於基準 hash
```

**Urban-Renewal（合併後 apps/web/）**
- headless 迴歸測試（Node stub-DOM）全綠；HTML 檔在瀏覽器實開一次。
- 真實段名 grep 零命中（同上指令）。

**BUILDER／governance 文件**
- 內文提到的路徑與指令，逐一確認存在／可跑（用 ls 與 --help，不憑印象）。
- CLAUDE.md ≤ 150 行。

## R6 數字的可信度分級（報數字前先分級）

- **A 級**：Core 算出、黃金測試鎖定 → 可直接報。
- **B 級**：Core 算出、無測試鎖定（如 valuation 基礎版）→ 報告時標「未校準」。
- **C 級**：自己心算／推估 → **禁止出現在任何交付物**。要嘛叫 Core 算，要嘛標「示意值，非計算結果」。

**正例**：報告寫「共負比 55.92%（案例A，L6 鎖定值）」。
**反例**：在說明文件裡自己算一個「大約 60%」當例子——弱模型算錯機率高，且違反 SSOT 精神。
