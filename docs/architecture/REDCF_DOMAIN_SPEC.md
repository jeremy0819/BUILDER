## PART 0 · 玩後觀察（Team Playtest）

### 🏗️ 都更專家視角：Core 比想像深，但下一層還沒長出來

**已具備的真實深度（給你肯定）：**
- **§162 逐層容積查核** — 正確處理樓梯間/梯廳逐層計算，這是多數 Excel 會錯的地方
- **§56 逐戶權變** — 權利變換到「逐戶」層級，這是機構級的細度
- provenance 溯源 + 三案例重算驗證 — 可稽核，SSOT 落地

**下一層的缺口（值錢的肉在這）：**
1. **容積獎勵建模** — 目前獎勵率是 input param。但獎勵是都更**最大價值槓桿**（案例A原型案未獲防災獎勵，每戶減損千萬）。獎勵不該只是一個數字，該是一組可拆解的規則。
2. **更新前價值係數** — §56 逐戶權變要算「權值比例」，就需要「更新前各戶價值」。這正是 B 系列的卡點（見 PART 3）。
3. **EV / 完工機率層** — 目前算的是名目財務。缺「期望值」（完工機率 × 折現）與「三方拆解」。這是 M4 Decision Engine 的事，但公式先寫在這（PART 1.3）。

### 🎮 遊戲設計師視角：三幕結構漂亮，但入口與資訊密度要收

**優點：**
- `輸入 → 推演 → 結算` 是乾淨的三幕劇，STAGE 01→02→03 有編號+箭頭，方向清楚。
- 「一塊地，一群人，一個真源」是好 slogan——一句話講完世界觀。
- STAGE 03 結算做成「可收藏的故事化復盤」= 好的留存設計。
- 沙盤把「共負比決定人心」變成可玩因果，這是整個產品的引擎。

**要修：**
- **入口稀釋**：首頁「延伸資源」塞了 6+ 連結（11 關指南/白皮書/簡報/V4 經典/GitHub）。遊戲設計上，這會讓新玩家的眼睛離開主迴圈。建議：主迴圈（三幕）視覺放大，延伸資源收進次選單。

## PART 1 · RE-DCF 核心領域模型規格

### 1.1 輸入參數（六層）

**① 基地／法規層**
- 基地面積（m²／坪）
- 使用分區 → 法定容積率、建蔽率
- 前面道路寬度（影響容積、開放空間獎勵、**更新前價值**）
- 模式：都更／危老／防災都更

**② 容積層（價值放大器）**
- 法定容積 = 基地面積 × 容積率
- **容積獎勵**（可拆解規則，非單一數字）：
  - 都更：時程、規模、綠建築、智慧建築、耐震、無障礙、防災、開放空間（各有 %，有上限）
  - 危老：基準 10% + 時程獎勵（逐年遞減）+ 規模，上限 1.3×基準（或 1.15×+外加）
- 容積移轉（TDR）
- 免計容積：陽台、梯廳、機電、雨遮（**銷坪比驅動**）

**③ 銷售層**
- 總樓地板面積、可銷售面積、公設比
- **銷坪比 = (允建容積 + 陽台免計) / (1 − 公設比) / 允建容積**
  （34% 公設比時穩定約 1.62–1.63）
- 房屋單價（萬/坪，可分樓層/朝向）、車位數 × 車位單價

**④ 成本層 → 共同負擔（條例 §51、權變辦法 §19）**
- 營建工程費（萬/坪，依結構/樓層）、拆除費
- 規劃設計費（建築師/都更顧問/估價師/地政士）
- 管理費（全案管理 %、更新會行政）
- 融資利息（建融/土融）、稅捐、銷售費用（代銷 %）
- 權利變換費用、都更規費

**⑤ 財務／時間層**
- 各階段時程（見 1.4 階段樹）
- 折現率 r、完工機率 p、土地自然增值率 g

**⑥ 更新前價值層（B 系列卡點）**
- 更新前各戶現況價值 → 決定權值比例（§56 分母）
- 由 **路寬帶 × 分區 × 建物型態 × 樓層** 決定（見 PART 3）

### 1.2 核心恆等式

```
更新後總價值 V = 地主分回總值 + 共同負擔
共同負擔 = V × 共負比 c
地主分回總值 = V × (1 − c)

個別地主分回 = 分回總值 × 權值比例ᵢ
權值比例ᵢ = 更新前價值ᵢ / Σ更新前價值   ← 需要 PART 3 係數
```

### 1.3 EV 三方拆解公式（M4 Decision Engine 用，公式先立此）

**參數**：p = ∏階段存活率（見 1.4）；df = 1/(1+r)^T

```
【名目層】
地主名目   = V(1−c) − V₀
實施者名目 = 收入 − 成本
   全案管理：收入 = V × 管理費率；成本 = 前期代墊 + 營運
   合建　　：收入 = 建商分回房地；成本 = 營建 + 土地
我方名目   = 依角色（營建管理費／開發利潤／投報）

【期望值層】
地主 EV   = p·df·[V(1−c)] − V₀·(1+g)^T·df
            （成功才拿分回；對照組=抱舊房以 g 自然增值）
實施者 EV = p·df·(管理費收入) − (1−p)·前期代墊沉沒 − df·營運
            （前期收費+低資本 → 常為正，失敗只賠代墊）
我方 EV   = 依角色套同框架

【判讀】
三方 EV 落差 = 整合風險；EV 為負的一方 = 破局引爆點
蛋黃區：V₀ 高 → 地主 EV ≈ 0/負 → holdout
蛋白區：容積算不動 → 實施者 EV 負 → 沒人做
```

### 1.4 開發階段樹（完工機率骨幹）

每階段帶「存活率 pᵢ」與「時間 tᵢ」（存活率為示意預設，須以實際案件校準）：

| # | 階段 | 存活率(示意) | 時間 | 主要風險 |
|---|---|---|---|---|
| 1 | 整合啟動 → 達門檻(都更80%/危老100%) | 0.65 | 1–3 yr | **最大過濾器**（holdout） |
| 2 | 門檻 → 報核（事業+權變） | 0.85 | 0.5 yr | 文件/選配爭議 |
| 3 | 報核 → 審議核定 | 0.80 | **2.34–3.74 yr** | 陳情/訴訟 |
| 4 | 核定 → 融資到位 | 0.92 | 0.5 yr | 核貸不足 |
| 5 | 融資 → 建照 | 0.95 | 0.5 yr | 圖審 |
| 6 | 建照 → 開工 | 0.95 | 0.3 yr | 拆除/騰空 |
| 7 | 開工 → 完工 | 0.85 | **3.5 yr** | 營造/財務/**爛尾**（資本額問題） |
| 8 | 完工 → 交屋結算 | 0.95 | 0.5 yr | 找補/登記 |

```
累積完工機率 = ∏pᵢ ≈ 0.34（從整合啟動起算）
→ 越早期，期望值折損越兇；已核定案後段 ∏ ≈ 0.77
```

### 1.5 敏感度變數清單（Tornado）

依「對結果影響」排序，供敏感度分析/龍捲風圖：

1. 房屋單價（營收最大槓桿）
2. **容積獎勵率**（面積最大槓桿）
3. 營建成本 /坪（成本最大槓桿）
4. **完工機率 p**（EV 殺手）
5. 審議/施工時程（折現驅動）
6. 共負比 c（分配驅動）
7. **更新前價值 V₀**（status quo / 權值分母）
8. 折現率 r
9. 土地自然增值率 g（status quo 對照）
10. 融資利率

## PART 2 · 三個對抗性合成案例（回歸測試）

> 把三種真實破局模式**參數化為合成案例**（零真實資料）。每個案例 = 一條回歸斷言：模型跑完必須吐出指定診斷，證明它「懂」都更，不只會算術。

### Case A · 內爆型（案例A・蛋黃區內爆原型）：蛋黃區偏好雪崩
```
參數：蛋黃區(V₀ 高)、都更 80% 門檻、觀望戶多、
      實施者=全案管理(低資本)
模型必須吐出：
  ✓ 地主 EV ≈ 0/負（V₀ 高，status quo 好）
  ✓ 實施者 EV 正 → 「實施者正、地主≈0」不對稱旗標
回歸斷言：偵測到 stakeholder EV 不對稱 → CAUTION（整合風險）
```

### Case B · 背信型（案例B・合建背信原型）：假權變真合建 hold-up
```
參數：合建模式、分配比例未定(空白)、前期關係專屬投資高
      (收購釘子戶+畸零地)、地價 T0→T1 翻倍
模型必須吐出：
  ✓ 地價敏感度極高
  ✓ 地價漲後「地主背信 EV > 履約 EV」的翻轉點
  ✓ 實施者關係專屬投資的套牢風險
回歸斷言：地價上漲情境下，標出實施者 hold-up 曝險
```

### Case C · 攔胡型（案例C・外部攔胡原型）：外部競標
```
參數：簽約前競爭、對手報「1坪換1坪+車位」、
      舊 8% 公設 vs 新 33% 公設
模型必須吐出：
  ✓ 名目分回 vs 室內實坪淨值的落差（公設比煙霧彈）
  ✓ overpay 對手的財務脆弱度 → 完工機率下降
回歸斷言：揭穿室內實坪縮水 + overpay→p 下降
```

## PART 3 · B 系列解鎖：更新前價值係數表（三張）

### ⚠️ 定位聲明（務必先讀）

> **這三張表是「訓練模式沙盤（B 系列）的合成預設係數」，方向正確、量值示意，用來讓遊戲行為擬真。**
> **它們不是估價值。** 真實案件的更新前價值，依《權利變換實施辦法》必須由**不動產估價師**以正式估價方法（比較法/收益法/成本法）評定。
> 你的架構把這件事設成核准紅線是對的——所以下面給你的是**有結構的起點**，供你（領域專家）核准/校準，而非讓 UI 憑空編。**任何真實案件請以估價師報告為準。**

|---|---|---|
| 巷弄 < 6m | 0.85 | 住宅巷內，可及性低 |
| 6–8m | 0.95 | |
| 8–15m | **1.00** | 基準 |
| 15–20m | 1.15 | 次幹道 |
| 20–30m | 1.30 | 幹道，商業潛力 |
| > 30m / 臨大道 / 角地 | 1.45 | 角地/大道另加成 |

| 建物型態 | 係數 | 說明 |
|---|---|---|
| 無電梯公寓（步登 4–5F） | **1.00** | 基準（都更主力標的） |

| 違建 / 增建部分 | 0.30–0.50 | 合法權益有限 |

> **第四維度提醒**：**樓層**應獨立成係數（1F 店面 >> 樓上住宅，1F 常 2–4×）。建議 B 系列加 `樓層係數`，否則店面戶會被低估。

### 組合公式

```
單戶更新前價值 = 基準單價 × 路寬係數 × 分區係數 × 建物型態係數 × 樓層係數 × 面積
權值比例ᵢ = 更新前價值ᵢ / Σ更新前價值
```

### 校準路徑（給你把「示意」變「可用」）

1. 拿 1–2 份**真實估價報告**（去識別化），反推各維度的實際權重帶。
2. 用實價登錄（同分區/同路寬帶）校準「基準單價」。
3. 三維度乘積結果，對照估價師的逐戶價值排序（**要順序一致**，量值容許帶狀誤差）。
4. 校準後的係數表，才從「遊戲預設」升格為「案件可用」——並仍標示「非估價值、僅供試算」。

### B1–B5 連動（解鎖後可一次做完）
- **B1** 可變戶數矩陣 12–80：每戶 = 路寬×分區×型態×樓層×面積 → 權值比例
- **B2–B5** 家族連動 + 沙盤逐戶依案件戶數：以 B1 的權值矩陣為基礎展開
