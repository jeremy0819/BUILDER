## P0 逐項執行說明

### 1. 自訂 70% 閘門已退場 ✅
- **1.1 檔**：健檢面板只顯示 Core `warnings[]`（`SHARED_COST_HIGH`／`SHARED_COST_LOW` 等 code＋message），本頁不自判。
- **1.0 舊檔**：共負比、銷坪比改為**純數值顯示**（不上紅黃燈），註明「合理帶依案型×投報模式而異（危老 25–45／都更全管 30–50／合建 35–55），由 Core 判斷」。
- 已知悉**紅黃燈會與過去不同**是預期行為（貴方門檻為實案校準值）；UI 免責文與空狀態說明同步改寫，內部使用者可從畫面文字直接理解變更原因。
- 驗證：全頁 grep「70% 具／70% 仍具」零命中。

### 2. 驗證器真升級 v1.1 ✅
- `schema_version >= 1.1` 時，`result.warnings`（陣列）／`computed_at`（字串）／`core_version`（字串）**列為必要**，缺任一即擋下並報具體欄位。
- **舊格式處理決定**（採貴方建議）：`1.0` 檔**不擋匯入**，健檢面板首行顯示「⚠️ 舊格式檔（schema 1.0）：未含 Core 權威健檢，建議用新版 RE-DCF 重新匯出」。
- 邊界測試：把 1.0 檔的 `schema_version` 改成 `"1.1"`（三欄缺）→ 正確擋下。
