# RE-DCF-Tool — 都更/危老前期評估工具

## 專案目的

Streamlit 互動式工具，將「建築坪效與前期評估」Excel 財務模型轉化為程式。
輸入土地與建築參數 → 自動完成容積查核、銷售坪效計算、開發評效分析，並偵測法規踩坑點。

開發者背景：建築學碩士 + 都更開發 PM，無正式程式經驗，正在學 Python。
程式碼要求：清晰、大量中文註解、變數可用中文、優先可讀性而非簡潔。

---

## 技術棧

| 套件 | 用途 |
|------|------|
| Streamlit ≥ 1.30 | 前端 + 後端，單一 `app.py` |
| pandas ≥ 2.0 | 樓層表格、CSV/Excel 解析 |
| plotly ≥ 5.18 | 儀表盤、瀑布圖、熱力圖 |
| openpyxl ≥ 3.0 | 讀寫 Excel（`make_template.py`） |

部署目標：Streamlit Community Cloud（免費，`main` 分支自動部署）。

**執行指令**：
```bash
pip install -r requirements.txt
streamlit run app.py        # 啟動 Web UI
python test_golden.py       # 黃金測試（每次改公式必跑）
python make_template.py     # 重新產生 Excel 對照範本
```

Windows 快速啟動：`啟動工具.bat`（呼叫 `streamlit run app.py`）。

## 計算架構（六層骨架）

| 層 | 名稱 | 關鍵公式 |
|----|------|---------|
| L1 | 輸入層 | 基地/容積/免計基準/成本/都更全案投報參數（sidebar） |
| L2 | 容積 | `FA = 基地使用 × 容積率`；`允建容積 = FA × (1+獎勵率) + 容積移轉` |
| L3 | 三項免計 | 梯廳逐層 %、安全梯（允建×15%）、陽台逐層 10%（**非** FA 總量） |
| L4 | 容積帳 | `計入容積 = 圖說計入 + 梯廳超出 + 陽台超出`；`餘量 = 允建 − 計入` |
| L4.5 | 銷售坪效 | `銷售坪 = (允建容積+陽台免計)×外皮係數 / 3.3058 / (1−公設比)`；銷坪比 1.58–1.68 正常 |
| L5 | 開發評效 | `評效 = 總銷 / 總成本`（>5 優良 / 2–5 可行 / <2 偏低） |
| L6 | 都更全案投報 | 總銷 → 共同負擔六大科目 → 共負比 / 地主分回 / 報酬率 |

**L6 共同負擔六大科目**（都市更新權利變換實施辦法）：

| 科目 | 計算方式 |
|------|---------|
| A 工程費用 | 營造費 + 設計監造（營造×5%）+ 工程管理（營造×3%） |
| B 管維費用 | A × 1%（竣工後設施維運） |
| C 權變費用 | 房地銷×1.5% + 戶數×拆補費 + 戶數×月租×月數 |
| D 貸款利息 | 土融利息 + 建融利息 |
| E 稅捐 | 房地銷×5%（營業稅）+ 總銷×0.1%（印花） |
| F 管理費用 | 總銷×5%（實施者服務費） |
| **合計** | `共同負擔 = A+B+C+D+E+F` |

## 六大踩坑點（程式須防呆）

1. **梯廳 §162-1 逐層**：是各層樓板 × 5% 或 8%，**不是** FA×8% 總量
2. **陽台超出必須補計入**：超出 10% 的部分計入容積，不可寫 0
3. **基地面積用使照面積**：非謄本面積（實案曾見謄本與使照差約 10%，用錯會墊高免計上限）
4. **三來源比對**：建築師/代書/建設公司數字需在「圖說驗核區」比對
5. **B1F 防空避難室排除**：排除樓地板合計（§117）
6. **安全梯 15% 基準是「允建容積」**：非 FA

## 建築師反饋收集協議（建築師說「有出入」時，先問這三個問題再改 code）

> 目的：不把症狀直接轉發給 AI——先把症狀解讀成具體的輸入/輸出差異，才能精準修正。

收到建築師反饋後，先收集以下資訊再開新對話：

```
Q1：他在哪個步驟卡住？
    → 是「找不到輸入欄位」、「數字算出來不對」、還是「結果格式看不懂」？

Q2：他的實際操作是什麼？
    → 例：他有沒有從 Excel 面積表匯入？他是手動輸入還是直接填圖說數值？
    → 他在建築師事務所用的欄位名稱叫什麼（梯廳還是走廊？陽台還是露台？）？

Q3：他期望看到什麼，我們工具給他看到什麼？
    → 把兩個數字或兩個畫面截圖附上。
```

拿到這三個答案後，才能精準告訴 AI 要改什麼。
