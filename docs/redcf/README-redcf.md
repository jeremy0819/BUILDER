# 🏗️ RE-DCF-Tool — 都更/危老前期評估工具

> 內部工具。輸入土地與**逐層**建築參數，自動完成容積查核（建築技術規則 §162）、銷售坪效、開發評效，並偵測六大法規踩坑點。  
> **圖說（面積表）為唯一真實依據**，工具逐層偵測免計超出並補計，不蓋過建築師圖說。

## ✨ 功能（v4）

### 容積查核（L1–L4）
- **逐層表格**：每層樓板 / 計容積 / 梯廳 / 安全梯 / 陽台實填，可勾選排除（B1F 防空避難室 §117）
- **Excel / CSV 一鍵匯入**面積表，自動對應欄名
- **容積量表 + 結論橫幅**：一眼看出是否超出允建容積
- **三項免計逐層查核**：梯廳 / 安全梯 / 陽台（含 1/8 投影），超出自動補計入容積

### 坪效與財務（L4.5–L5）
- **銷售坪效推導**：允建容積 → 室內坪 → 銷售坪 → 銷坪比（正常 1.58–1.68）
- **開發評效快篩**：總銷 / 總成本（> 5 優良 / 2–5 可行 / < 2 偏低）
- **雙軸敏感度熱力圖**：公設比 × 售價

### 都更全案投報（L6）
- **共同負擔六大科目**（都市更新權利變換實施辦法）：工程 A / 管維 B / 權變 C / 利息 D / 稅捐 E / 管理 F
- **地主分回 / 共負比 / 報酬率**
- **售價 × 營造敏感度熱力圖**
- 另附 `都更全案投報_對照範本.xlsx` 供開啟舊 Excel 坪效分析表逐格對照

### UI（v4 新）
- 精簡 **4 欄 KPI**：允建容積 / 容積餘量 / 銷坪比 / 投報率
- **5 個 Tab**：① 容積查核　② 坪效分析　③ 財務評效　④ 都更全案投報　⑤ 報告匯出
- **Sidebar 雙欄輸入**，捲動高度縮減約 40%
- **全域 CSS**：Metric 卡底色、Tab 膠囊樣式、表頭底色、頁面留白
- **一頁報告匯出**（Markdown / CSV）

## 🧮 核心計算邏輯（§162，全部逐層）

| 層 | 計算 |
|----|------|
| L2 基準容積 FA | `(基地面積 − 人行廣場) × 容積率` |
| L2 允建容積 | `FA × (1 + 獎勵率) + 容積移轉` |
| L3 梯廳超出 | `Σ 各層 max(0, 梯廳 − 樓板 × 梯廳免計%)` |
| L3 陽台超出（免計%） | `Σ 各層 max(0, 陽台 − 樓板 × 陽台免計%)` |
| L3 陽台超出（1/8投影） | `Σ 各層 max(0, 陽台 − 樓板 × 1/8)` |
| L3 安全梯上限 | `允建容積 × 15%` |
| L4 計入容積(修正後) | `計入容積(圖說) + 梯廳超出 + 陽台超出` |
| L4 容積餘量 | `允建容積 − 計入容積(修正後)`（< 0 = 超出需調整）|
| L4.5 銷售坪數 | `(允建容積 + 陽台免計) × 外皮係數 / 3.3058 / (1 − 公設比)` |
| L6 共同負擔 | `A工程 + B管維 + C權變 + D利息 + E稅捐 + F管理` |

> ⚠️ **逐層法才是正解**：陽台/梯廳超出必須各層樓板×% 判斷，用「總量 vs FA×%」會把超出量高估約 10 倍（實案驗證，紀錄私有）。

## ⚠️ 六大踩坑點

1. **梯廳 §162-1 逐層**：各層樓板 × 5% 或 8%，**不是** FA × 8% 總量
2. **陽台超出必須補計入**：超出 10% 的部分計入容積，不可寫 0
3. **基地面積用使照面積**：非謄本面積（實案曾見兩者差約 10%，用錯會墊高免計上限）
4. **三來源比對**：建築師 / 代書 / 建設公司數字需「圖說驗核區」比對
5. **B1F 防空避難室排除**：排除樓地板合計（§117），逐層表格取消勾選
6. **安全梯 15% 基準是「允建容積」**：非 FA
